# Support

## Community support

- Open a GitHub issue for bug reports and feature requests.
- Include your runtime profile and environment details.

## Direct contact

Email: `support@local.invalid`

## Self-hosted help checklist

When asking for help, include:

- OS and container/runtime versions
- your `APP_MODE`, `AUTH_MODE`, and `BILLING_MODE`
- relevant backend/frontend logs
- steps to reproduce from a fresh clone
